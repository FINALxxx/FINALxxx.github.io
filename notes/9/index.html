# 树、二叉树、森林的转换到哈夫曼树

本节对应紫书 8.4.3，并做了拓展学习

# 树与二叉树的互相转换

先看视频：

树 -> 二叉树口诀：

  1. 加线：兄弟之间加一连线（兄弟相连）
  2. 抹线：对每个结点，除了左孩子，去掉其与其他孩子的联系（立长子）
  3. 旋转

二叉树 -> 树口诀：

  1. 加线：若 q 结点是 p 结点的左孩子，那么 q 的右孩子 a、a 的右孩子 b...（以此类推，直到没有右孩子）都与 p 结点相连（长子右孩连父亲）
  2. 抹线：去掉原二叉树中每个结点与左孩子的联系（去长子）

# 二叉树与森林的转换

还是先看视频：

我们在思考这个方法时可以将每个二叉树抽象成 二叉树的根节点 ，那么森林其实就是很多个结点，也就可以组合成一棵二叉树。

当然，因为这个思考方式只是将二叉树抽象成根结点，便于记忆下面的口诀，实际上根节点的子树都被忽略了，所以这些结点不能直接组合成新二叉树。

森林 -> 二叉树口诀：

  1. 转换：将森林中每棵树都转换成二叉树（树变二叉）
  2. 加线：将每棵二叉树的根节点相连（根相连）
  3. 旋转

二叉树 -> 森林口诀：

  1. 抹线：抹去二叉树中根节点 p 与它右孩子 q 的连线、q 和它右孩子 a 的连线....（以此类推，直到没有右孩子），使一棵二叉树分成几棵孤立的二叉树（去右孩线）
  2. 转换：将每棵二叉树都转换成二叉树（再还原）

# 哈夫曼树构造 -- 最优二叉树 Huffman

最优二叉树：哈夫曼树使得存储字母的字节数最少（还就那个证明不来，看不懂刘汝佳的证明），因此在贪心思想中经常被使用到

叶子：目标的字母

叶子权值：这里每个叶子的权值是字母的频度（在文章中出现次数）

边值：在生成哈夫曼密码和解密时，假定（只是假定，可以换）每个结点的左边为 0、右边为 1，于是向左走为 0，向右走为 1

这里假设已经对哈夫曼树已经有基础的了解（如果没有可以参看王卓老师的系列视频），先看视频：

构造口诀：

  1. 构造森林全是根
  2. 选用两小造新树（可以选用 priority_queue）
  3. 删除两小填新人
  4. 重复 2、3 剩单根

性质（应该不怎么用到，就不解释了）：

  1. 贪心选择性质
  2. 最优子结构性质
