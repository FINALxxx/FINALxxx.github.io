# 用回溯法解决 8 个皇后的爱恨情仇

“别打了！别打了！你们这样是打不死的！”

本节内容对应紫书 7.4.1

无论是全排列生成还是子集生成，都有两种方法：直接枚举和递归枚举，直接枚举简单但速度极慢，递归枚举要好很多。
回溯法是对递归枚举的一种补充，在递归的同时检查，比如使用 vis [] 来存储 当前递归的路径 ，一旦递归后发现当前结点不满足题意，就可以撤回到上一结点去，同时在 vis [] 中将当前结点删除。

当然使用前提是这个题不用把所有结点都探索一遍，只需要一条从某点到另一点的路径就行。

回溯法应用范围 很广，只要树的深度不太大，每个结点的度也不太大，基本就可以用

# 直接枚举

有两种方法：

  1. 生成 64 个格子的子集，遍历所有子集，当子集 A 中有 8 个元素且每个元素的攻击范围中没有其他元素。（光是遍历所有子集就可能有 264 种，太慢了，还没遍历完可能皇后都被打没了）

  2. 从 64 个格子中选 8 个格子出来，再判断攻击范围（C648=4.426×109C^8 _{64} = 4.426 \times 10^9C648​=4.426×109，种类比方法一少了很多，但是依然不算少）

# 递归遍历

有二种方法，都是在模拟从上往下逐行地放皇后：

  1. 非回溯法：用二维数组 vis [x][y] 记录当前结点 (x,y) 是否在已经落下的棋子的攻击范围中。用 cur 记录当前要落棋的行数，对每一行来说，有 8 种落棋的方式，因此用一个 for 循环遍历 8 次，每次先进行判断， 用 if 判断放下的棋子是否在其他皇后的攻击范围中 ，如果不在，则落棋（在 vis [x][y] 中加入该结点），并传入 cur+1 进行递归（到下一行）。稍推荐

  2. 回溯法：用二维数组 vis [i][j] 记录已经落下的棋子 j 所占用的列（i=0）、主对角线 j 是否有棋子（i=1）、副对角线 j 是否有棋子（i=2）。同样用 for 遍历 8 次，每次先下棋在 vis [x][y] 中加入列数、主对角线参数、副对角线参数），然后传入 cur+1 进行递归， 递归的下一行一定要将 vis 重置为 0 ，这样，在回溯的时候才可以在 vis 中删掉撤回的棋子。更推荐

判断也可以放到递归出口那边写，但是这样的话，要先递归下一个结点才能开始判断当前结点是否可行。

  1. 非回溯 / 回溯法 + ok 变量判断法：基本与上面相同，但是这种方法可以判断的方式更多样。用一个 ok 变量来指示当前这个结点是否可行，每次遍历时通过额外的一次判断（不限于是否遍历过、是否过界等）得到 ok 的值，填数时只需判断当前数字是否 ok。较推荐

# 题外话：棋盘类型的方格中的主对角线和副对角线

主对角线 = 右斜线；副对角线 = 左斜线
对棋盘上 每一个 点 (x,y) 来说， 同一个主对角线上所有点的 x-y 的值相同， 同一个副对角线上所有点的 x+y 的值相同

因此可以用一个 vis1 [x-y] 保存 同一个 主对角线上是否 有一个或多个 棋子，防止主对角线出现负数，所以后面可以加个 n，用 vis1 [x-y+n] 保存；副对角线用 vis [x+y] 保存。
