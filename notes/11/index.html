# 紫书上暂时忽略的一些知识点

# 前缀和

i 元素的一维前缀和 / 后缀和：假设有一个一维数组，其中有一个元素下标是 i。那么 i 之前的元素的值的总和叫 i 元素的一维前缀和，i 之后的元素的值的总和叫 i 元素的一维前缀和（高维自己去查）

# 线段树

用数组当作树，用于求区间和（求前缀和、后缀和）或者模拟区间修改（分田、涂墙）

有两种线段树：

  1. 只进行 单点修改 的简单线段树
  2. 进行 区间修改（或者区间修改和单点修改混合） 的复杂线段树，需要打 <lazy 标记>

叶子结点内容：1. 从区间 1 到区间 n 的序号（不是数组下标） 2. 结点的值

非叶子结点内容：1. 左右儿子的值之和、差、积等（用于实现区间和等用途） 2. 左右儿子的区间和

# 复杂线段树

建树：同上，我就不写了

单点修改：同上

区间修改 下标 [a,b] 的所有值：
从根节点往下 dfs。

  1. 如果要修改的区间完全覆盖该区间就直接更新这个大区间结点的值，打上 lazy 标记表示只更新到这个结点，并不再往它的子结点探索

  2. 如果没有完全覆盖，就往当前的结点的子结点探索，但是如果当前结点有 lazy 标记，将 lazy 标记下传到下一层子结点并将该结点的区间更新（因为要修改更下面一层的结点，即继续查询子结点，又因为 探索到哪儿就更新到哪儿，没探索到就不更新，那么在查询的时候就更新 lazy 标记 ）

  3. 如果没有完全覆盖 且没有 lazy 标签 ，往当前的结点的子结点探索，如果查询区间和左儿子有交集（说明查询的区间中有些结点在左儿子区间中），就往下搜索左儿子

  4. 同第 3 点，如果查询区间和右儿子有交集（说明修改的区间中有些结点在右儿子区间中），就往下搜索右儿子

区间查询 下标 [a,b] 的所有值：
从根节点往下 dfs。

  1. 如果要查询的区间完全覆盖该区间就直接返回这个大区间结点的值，不再往它的子结点探索（不像单点修改那样必须找到目标结点才修改）
  2. 如果没有完全覆盖，往当前的结点的子结点探索，但是又如果当前结点有 lazy 标记，所以将 lazy 标记下传到下一层子结点并将该结点的区间更新（探索到哪儿就更新到哪儿，没探索到就不更新）
  3. 如果没有完全覆盖 且没有 lazy 标签 ，往当前的结点的子结点探索，如果查询区间和左儿子有交集，就往下搜索左儿子
  4. 同第 3 点，如果查询区间和右儿子有交集，就往下搜索右儿子
  5. 最后合并数据
    <lazy> 标记的作用 使大部分结点在区间修改时不用修改，延迟到查询的时候再修改。查询时，查询到哪儿就更新到哪儿，lazy 用于提示该结点已更新，但它的子结点需要更新，但是现在暂时没被查询到就不用更新。

# ST 表

采用倍增思想，用一个二维表在 O (1) 内在线查询 [l,r] 区间内的最值，有效解决 RMQ 问题

F [i][j]：从表中下标为 i 开始的一个区间 A，这个区间的长度是2j2^j2j，那么这个区间的终点的下标为i+2ji+2^ji+2j。F [i][j] 表示区间 A 中的最值。

建表：

  1. 递推公式：F [i][j]=maxF\[i\]\[j-1\], F\[$i+2({j-1)$][j-1]}
    也就是将一个区间按当前的步进长度 j（从 1 到 k）进行分割，分成多个子区间，再分别求最值，j 越大，单个区间就越大，被分割出来的区间个数就少。

  2. k 的求值：从原点开始找表的终点下标 n，按倍增的思想，将在2k2^k2k 到2k+12^{k+1}2k+1(闭区间) 中找到终点，于是反过来就知道，k=⌊log⁡2n⌋\lfloor \log_{2}n \rfloor⌊log2​n⌋。即在第 k 次分割时，因为 j=k，j 达到最大，于是单个区间的长度是整个表的长度，相当于没有分割。
